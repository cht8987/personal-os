# Schema: 任务（Task）

> 三件套之二（Retain）。一切待办的标准形态。

## 文件命名

`01-Inbox/task-YYYY-MM-DD-HHmm-<slug>.md`（入流时）
归档后 → `02-Memory/dynamic/ops-log/tasks.md`（集中清单，便于移动端打勾）

## 标准格式（集中清单内的条目）

```markdown
- [ ] 给服务器换证书 ⏫ 📅 2026-06-15 #project/bank-os ^task-20260610-2130
```

字段说明（兼容 Obsidian Tasks 插件语法）：

| 片段 | 含义 | 必填 |
| --- | --- | --- |
| `- [ ]` | 状态：`[ ]` 待办 / `[x]` 完成 / `[-]` 取消 | ✅ |
| 正文 | 动词开头的一句话，≤ 30 字 | ✅ |
| `⏫/🔼/🔽` | 优先级 高/中/低 | 可选，默认中 |
| `📅 YYYY-MM-DD` | 截止日 | 可选 |
| `#tag` | 项目/领域标签 | 可选 |
| `^task-…` | 唯一块引用 ID（ingest 自动生成） | ✅ |

## 提取规则（供 ingest AI 使用）

- 文本含明确的"要做/记得/明天/下周/别忘了 + 动作" → 提取为 Task。
- 模糊愿望（"以后有空看看"）不算 Task，留在日记随想里。
- 日期相对表达必须换算为绝对日期（"明天" → 实际 YYYY-MM-DD）。
